# 小白个性化训练、重量记录与趋势蛋糕：设计规范

## 目标

让女性健身小白在少量清晰输入后，得到贴合现实的训练动作、实际器械重量选择与下一次调整建议。界面只说“今天做什么、先拿多重、做完点哪里”；内部采用保守、可解释的规则。身高、体重、旧伤说明和训练便签不用于给用户贴胖瘦类别，也不直接决定训练重量。

## 范围

### 首版包含

1. 在“我的 → 训练偏好”内追加分步资料设置：
   - 目标：建立习惯、塑形、减脂＋饮食。
   - 身高（cm）与当前体重（kg）；体重默认每周记录一次。
   - 训练基础：从未练过、偶尔练、能稳定训练。
   - 场地与器械；器械可多选，哑铃重量支持徒手、0.5/1/1.5/2/3/4/5/7.5/10kg+、自定义档位与可调范围。
   - 旧伤/常不适自由说明，并选择会不舒服的动作：蹲、弯腰、推举、跳跃、久坐后起身。
   - 安全确认：无需要暂停训练的情况 / 近期受伤、医生限制或其他需要先咨询专业人士的情况。
   - 仅当目标为“减脂＋饮食”时收集外卖频率、早餐、蛋白质习惯与忌口；输出盘餐法、外卖/便利店选择和蛋白质提醒，不计算热量。
2. 每日计划根据目标、训练基础、器械、动作限制、今日状态与经期调整呈现具体动作卡。
3. 每张动作卡记录实际器械、实际重量/阻力、完成组次、感受与可选便签。
4. 下次同一动作显示最近一次记录与最近三次对比，并在实际拥有的器械档位中给出保守建议。
5. “蛋糕趋势”作为个人趋势与习惯累积的鼓励性可视化；问号解释其仅显示个人趋势，不评价体型、体脂或外貌。

### 首版不包含

- 从身高、体重或围度推断体脂、胖瘦类别或“身材阶段”。
- 卡路里、宏量营养目标、极端饮食或医学营养处方。
- 自动解析旧伤自由文本、诊断、康复治疗承诺或 AI 医疗结论。
- 排卵/易孕推断、可穿戴设备接入、视频动作识别。

## 用户体验

### 资料设置

资料在“我的 → 训练偏好”中分步展示，不阻断首次生成计划。每一步可以返回或稍后补充；未填写时使用保守默认方案，并明确提示“补充器械和经验后会更贴合”。

身高、体重、旧伤说明、动作限制和便签均默认为仅本机。它们不得随通用快照上传或恢复；未来跨设备同步需独立、明确的身体资料同步同意。

### 今日动作卡

动作卡使用小白语言，不展示 RIR、RM、百分比强度或复杂阶段术语。示例：

> 单臂哑铃划船：2 组 × 8 次
>
> 先拿：你已有的 3kg
>
> 合适的感觉：最后两次有点费劲，但动作仍稳
>
> 若肩颈不舒服：改为弹力带划船或减少范围

动作卡必须展示：今日动作、目标组次、上次实际记录、今天建议、可用器械档位、替代动作和不舒服时的停止提示。

### 训练记录与便签

每个动作允许用户一键确认实际器械和实际重量，再选择：太轻松、刚刚好、有点难、不舒服。便签最长 120 字，位于反馈下方，默认折叠为“留一句给下次的自己（选填）”。

动作历史默认只展示最近三次，格式为“日期 · 器械/重量 · 组次 · 感受”，让用户直接比较，而不是阅读图表或算法。

### 蛋糕趋势

蛋糕的层数/装饰表示个人习惯与趋势的累积，来源仅为每周体重记录、训练完成和饮食习惯。它不映射“低体脂、正常、偏胖”等外形分类。问号文案：

> 这是你和自己的趋势对比，不是体脂测试，也不会决定你今天该练多重。

## 数据模型

所有对象先存在 localStorage；身体资料相关对象不加入 `syncSnapshot` 的通用键列表。

```js
trainingProfile = {
  goal: 'habit' | 'shape' | 'fat_loss_food',
  heightCm: number | null,
  experience: 'new' | 'occasional' | 'consistent',
  places: string[],
  equipment: {
    bodyweight: boolean,
    bands: boolean,
    dumbbellKg: number[],
    adjustableDumbbell: { minKg: number, maxKg: number } | null,
    kettlebellKg: number[],
    gymMachines: string[],
  },
  discomfortNote: string,
  avoidMovements: string[],
  safetyFlag: 'none' | 'recent_injury_or_clinician_restriction',
  dietHabits: { takeout: string, breakfast: string, protein: string, restrictions: string },
};

bodyTrendHistory = [{ date: 'YYYY-MM-DD', weightKg: number }];

exerciseSessionHistory = [{
  date: 'YYYY-MM-DD',
  exerciseId: string,
  exerciseName: string,
  equipment: string,
  loadKg: number | null,
  sets: [{ plannedReps: number, completedReps: number }],
  feedback: 'too_easy' | 'just_right' | 'somewhat_hard' | 'uncomfortable',
  note: string,
}];
```

输入校验：身高、体重、重量、组数和次数均为有限正数且设合理上限；哑铃档位排序去重；便签和旧伤说明进行长度限制并不上传到日志或控制台。

## 决策规则

规则严格按以下优先级执行：

1. **安全中止：** 用户明确选择“近期受伤、医生限制或需要先咨询专业人士”，或当天出现既有红旗、明显疼痛或头晕时，不生成加量建议；显示暂停训练和咨询专业人士提示。自由文本本身不作自动病情判断。
2. **今日状态：** 现有的疲劳、睡眠、疼痛、经期红旗/恢复规则先决定正常、轻量、恢复或暂停。
3. **动作限制：** 若用户选择会不舒服的动作，过滤对应动作模式；用低冲击、较小范围或徒手/弹力带替代。自由文本只显示给用户本人，不自动推断病因。
4. **器械与目标：** 仅选择用户确认拥有的器械和重量。首次无历史记录时推荐徒手或最小可用档位作为试组，不给所有用户固定公斤数。
5. **渐进调整：**
   - 所有计划组次完成、动作稳定且反馈“太轻松”时，下次可上调至下一个可用重量档位。
   - “刚刚好”保持当前档位。
   - “有点难”保持或降至上一档，避免自动加量。
   - “不舒服”不加重量，优先替代动作，并显示停止/咨询提示。

体重、围度、蛋糕趋势和训练目标绝不作为单独的负荷公式输入。体重不决定哑铃重量，旧伤文本不产生医疗建议。

## 依据与文案边界

- 成人应每周至少两天覆盖主要肌群的力量训练；小白从可持续频率与动作质量开始。[WHO](https://www.who.int/initiatives/behealthy/physical-activity)
- 健康成人的新手阻力训练可从可控的 8–12 次范围开始，方案要结合目标、能力和训练状态。[ACSM position stand](https://pubmed.ncbi.nlm.nih.gov/11828249/)
- 内部用“剩余余力”思路帮助选择当天负荷，但 UI 改写为小白可理解的“最后两次有点费劲、动作仍稳”。[NASM RIR guide](https://blog.nasm.org/reps-in-reserve)

所有训练文本使用“可以考虑”“如不适就停止”，不承诺治疗、减重速度或身体外形结果。

## 验收与测试

1. 新手能在训练偏好设置中完成资料，也能跳过后先使用保守计划。
2. 有 2kg、3kg、4kg 哑铃时，系统只显示这些档位；无器械时不显示哑铃加重建议。
3. 首次动作记录保存实际重量、组次、反馈和便签；下次同一动作显示最近一次与三次历史。
4. “太轻松”只在完整完成且无不适时推进到下一现有档；“有点难”与“不舒服”永不自动加重。
5. 选择“蹲会不舒服”时，计划不再安排深蹲类动作，并展示替代动作和非诊断提醒。
6. 周体重趋势更新蛋糕，但不会显示胖瘦/体脂等级，且不会改变当天动作重量。
7. 身体资料、旧伤说明和便签默认不包含在通用云端快照；单独同意前不会读取或写入云端。
8. 现有经期同意门、计划生成、记录、日历、登录与冒烟测试全部回归通过。
